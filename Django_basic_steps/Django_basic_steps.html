Django Basic Steps

Virtual Environment

  1. Create Virtual Environment

    cmd: ...path\Project>python –m venv virtual

    "virtual" is a name, It can be "anything"

  2. Activate Virtual Environment

    cmd: ...path\Project>virtual\Scripts\activate

Django Installation

  1. Install

    cmd: ...path\Project>pip install Django

  2. Verify Installation

    cmd: ...path\Project>django-admin --version

  3. Auto-generate requirements.txt

    (option1)cmd: ...path\Project>pip freeze > requirements.txt

    (option2)cmd: ...path\Project>pip install pipreqs
    ...path\Project>pipreqs /path/to/project

  4. Auto-install from requirements.txt

    cmd: ...path\Project>pip install -r requirements.txt

Django Create Project

  1. Create Project's Container

    cmd: ...path\Project>django-admin startproject containerName

    "containerName" can be anything, inside it are the project files

  2. Check Temporary Server

    cmd: ...path\Project>py manage.py runserver

  3. Create App

    cmd: ...path\Project>py manage.py startapp appName

    "appName" can be anything. An app can be in multiple projects.

    Inform Django - Models Create/Change

    cmd: ...path\Project>py manage.py makemigrations appName The same applies when Models do not exist yet

    Preview Database Details

    cmd: ...path\Project>py manage.py sqlmigrate appName fileNumber "fileNumber" is the number in the database filename, e.g. appName\migration\0001_initial.py

    Check for errors

    cmd: ...path\Project>python manage.py check

    Apply Model Changes to Database

    cmd: ...path\Project>python manage.py migrate

    Three step guide
    • Change your models (in models.py).
    • Run python manage.py makemigrations to create migrations for those changes
    • Run python manage.py migrate to apply those changes to the database

    The reason that there are separate commands to make and apply migrations is because you’ll commit migrations to your version control system and ship them with your app
